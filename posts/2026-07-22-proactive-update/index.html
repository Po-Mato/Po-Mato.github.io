2. Gateway м„ұлҠҘ лӘЁлҚёмқҳ 3лҢҖ м°Ёмӣҗ

ASTP Gatewayмқҳ м„ұлҠҘмқҖ м„ё к°Җм§Җ м¶•м—җм„ң м •мқҳлҗңлӢӨ:

2.1 Throughput (RPS)

Stateless MCP нҷҳкІҪм—җм„ң Gatewayмқҳ ThroughputмқҖ WAL Write Latencyм—җ мқҳн•ҙ м ңн•ңлҗңлӢӨ. к°Ғ мҡ”мІӯмқҖ Gatewayм—җ лҸ„м°©н•ҳл©ҙ WALм—җ кё°лЎқлҗҳкі , CRDT DeltaлЎң ліҖнҷҳлҗң нӣ„ DRT(Delta Routing Table)м—җ л”°лқј peer GatewayлЎң м „нҢҢлҗңлӢӨ.

Request вҶ’ [WAL Append] вҶ’ [CRDT Delta Serialize] вҶ’ [DRT Lookup] вҶ’ [Peer Send]
           t_wal          t_serialize                    t_send

мҙқ мІҳлҰ¬ мӢңк°„: T_total = max(t_wal, t_serialize + t_send)

м—¬кё°м„ң t_walмқҙ м§Җл°°м ҒмқҙлӢӨ. WALмқҖ fsync кё°л°ҳмқҳ лӮҙкө¬м„ұ ліҙмһҘмқҙ н•„мҡ”н•ҳлҜҖлЎң (PIPA к°җмӮ¬ мҰқм Ғ мҡ”кө¬мӮ¬н•ӯ), disk I/Oк°Җ лі‘лӘ©мқҙ лҗңлӢӨ.

2.2 Latency (P50/P95/P99)

LatencyлҠ” лӢӨмқҢ кө¬м„ұ мҡ”мҶҢлЎң л¶„н•ҙлҗңлӢӨ:

L_total = L_network(tcp) + L_tls + L_request_parse + L_wal + L_serialize + L_drt + L_send

Stateless MCPм—җм„ң L_tlsлҠ” connection reuseлЎң л¬ҙмӢң к°ҖлҠҘн•ң мҲҳмӨҖмқҙм§Җл§Ң, мІ« мҡ”мІӯмқҳ connection setup л№„мҡ©мқҖ м—¬м „нһҲ мЎҙмһ¬н•ңлӢӨ. ASTPм—җм„ңлҠ” HTTP/2 multiplexingкіј connection warm-up poolлЎң мқҙ л№„мҡ©мқ„ л¶„мӮ°н•ңлӢӨ.

н•өмӢ¬ л°ңкІ¬: Stateless м „нҷҳмңјлЎң к°ҖмһҘ нҒ° Latency к°ңм„ мқҙ мқјм–ҙлӮҳлҠ” кө¬к°„мқҖ L_drtк°Җ м•„лӢҲлӢӨ (ASTPлҠ” мқҙлҜё л¶„мӮ° мІҳлҰ¬). мҳӨнһҲл Ө sticky session м ңкұ°лЎң мқён•ң load imbalance н•ҙмҶҢк°Җ P99 LatencyлҘј 40% мқҙмғҒ к°ңм„ н•ңлӢӨ.

2.3 Consistency (DRT Sync Delay)

ASTPмқҳ ConsistencyлҠ” DRT(Delta Routing Table)мқҳ Sync DelayлЎң мёЎм •лҗңлӢӨ. DRTлҠ” к°Ғ Gatewayк°Җ peerлЎңл¶Җн„° л°ӣмқҖ CRDT DeltaлҘј м Ғмҡ©н•ҳлҠ” ClockмңјлЎң, мқҙ delayк°Җ нҒҙмҲҳлЎқ Stale Read к°ҖлҠҘм„ұмқҙ лҶ’м•„м§„лӢӨ.

Sync_Delay(t) = max(Clock_i(t) - Clock_j(t)) for all i, j in Gateway Mesh

лІӨм№ҳл§ҲнҒ¬м—җм„ң мёЎм •н•  н•өмӢ¬ л©”нҠёлҰӯмқҖ Sync_Delayмқҳ л¶„нҸ¬лӢӨ. к°•н•ң мқјкҙҖм„ұмқ„ мҡ”кө¬н•ҳлҠ” PIPA к°җмӮ¬ мҰқм Ғ мӢңлӮҳлҰ¬мҳӨм—җм„ңлҠ” Sync_Delay < 50msлҘј SLOлЎң м„Өм •н•ңлӢӨ.

4. Capacity Planning лӘЁлҚё: Little's Law + M/G/1 Queue

4.1 GatewayлҘј QueueлЎң лӘЁлҚёл§Ғн•ҳкё°

ASTP GatewayлҠ” Stateless MCP мң„м—җм„ң лҸҷмһ‘н•ҳлҜҖлЎң, к°Ғ Gateway мқёмҠӨн„ҙмҠӨлҘј M/G/1 Queue (Poisson arrival, General service time, 1 server)лЎң лӘЁлҚёл§Ғн•  мҲҳ мһҲлӢӨ.

Little's Law: L = О» Г— W

  • L: In-flight requests (лҢҖкё° мӨ‘мқё мҡ”мІӯ мҲҳ)
  • О»: Arrival rate (RPS)
  • W: Average time in system (latency)

M/G/1 Pollaczek-Khinchine кіөмӢқ:

W = (О» Г— E[SВІ]) / (2 Г— (1 - ПҒ)) + E[S]

where:
  ПҒ = О» Г— E[S] (utilization)
  E[S] = нҸүк·  service time
  E[SВІ] = service timeмқҳ 2м°Ё лӘЁл©ҳнҠё

мқҙ кіөмӢқмқҖ Gatewayмқҳ м„ңл№„мҠӨ мӢңк°„ л¶„мӮ°(variance)мқҙ Latencyм—җ м–јл§ҲлӮҳ м№ҳлӘ…м Ғмқём§Җ м •лҹүнҷ”н•ңлӢӨ. CRDT Delta м§Ғл ¬нҷ” нҒ¬кё°к°Җ л“ӨмӯүлӮ мӯүн• мҲҳлЎқ (н•ңкөӯм–ҙ н…ҚмҠӨнҠёк°Җ нҸ¬н•Ёлҗң Delta vs м§§мқҖ JSON Delta), E[SВІ]к°Җ м»Өм§Җкі  лҢҖкё°м—ҙ Latencyк°Җ нҸӯл°ңн•ңлӢӨ.

4.2 мӢӨм „ Capacity Equation

RequiredNodes = ceil(О»_target / О»_single_node Г— replication_factor)

where:
  О»_single_node = min(1/E[S] Г— CPU_limit, IOPS_limit / IOPS_per_request)
  replication_factor = 3 (ASTP Gateway Mesh кё°ліёк°’)

мӢӨм „м—җм„ңлҠ” CPU н•ңкі„мҷҖ IOPS н•ңкі„ мӨ‘ лӮ®мқҖ мӘҪмқҙ кІ°м •м Ғ мҡ”мҶҢк°Җ лҗңлӢӨ.

WALмқҙ лі‘лӘ©мқј л•Ң:

function calculateWALCapacity(config: WALConfig): CapacityEstimate {
  const fsyncLatency = config.isNVMe ? 0.01 : 0.5; // ms (NVMe vs SSD)
  const batchSize = config.fsyncBatchSize; // records per fsync
  const maxWritesPerCore = 1000 / (fsyncLatency / batchSize); // writes/sec/core

  const throughputPerCore = maxWritesPerCore * config.avgRecordSize;
  const ioBandwidth = config.diskWriteBandwidth; // MB/s

  return {
    maxRPS: Math.floor(maxWritesPerCore * config.cores),
    ioBound: (throughputPerCore * config.cores) > ioBandwidth,
    bottleneck: (throughputPerCore * config.cores) > ioBandwidth
      ? 'disk_bandwidth'
      : 'wal_fsync_latency'
  };
}

мқҙ кёҖм—җм„ң мқҳлҸ„м ҒмңјлЎң лӢӨлЈЁм§Җ м•Ҡкұ°лӮҳ н•ңкі„лҘј мқём •н•ҳлҠ” л¶Җл¶„:

  1. мӢӨн—ҳмӢӨ лІӨм№ҳл§ҲнҒ¬мҷҖ мӢӨм „ нҷҳкІҪмқҳ м°Ёмқҙ: Section 7мқҳ мҳҲмғҒ кІ°кіјлҠ” нҶөм ңлҗң нҷҳкІҪ(L4 switch + dedicated NVMe + лҸҷмқј лҰ¬м „) кё°мӨҖмқҙлӢӨ. мӢӨм ң нҒҙлқјмҡ°л“ң нҷҳкІҪ(кіөмң  tenancy, cross-region latency, egress cost)м—җм„ңлҠ” кІ°кіјк°Җ нҒ¬кІҢ лӢ¬лқјм§Ҳ мҲҳ мһҲлӢӨ.

  2. M/G/1 лӘЁлҚёмқҳ н•ңкі„: Gatewayмқҳ Service TimeмқҖ CRDT Delta Mergeм—җм„ң л°ңмғқн•ҳлҠ” Lock Contention л•Ңл¬ём—җ strictly independentк°Җ м•„лӢҲлӢӨ. Heavy tail мғҒнҷ©м—җм„ңлҠ” M/G/1 мҳҲмёЎмқҙ 20~30% кіјмҶҢнҸүк°Җлҗ  к°ҖлҠҘм„ұмқҙ мһҲлӢӨ. G/G/1мқҙлӮҳ Simulation-based м ‘к·јмқҙ лҚ” м •нҷ•н•  мҲҳ мһҲлӢӨ.

  3. н•ңкөӯм–ҙ Delta Size нҺён–Ҙ: Section 5.3м—җм„ң м–ёкёүн–Ҳл“Ҝмқҙ, н•„мҲҳ л©”нғҖлҚ°мқҙн„°(clock vector, gatewayId л“ұ)к°Җ Deltaмқҳ м§Җл°°м Ғ мҡ”мҶҢмқҙлҜҖлЎң н•ңкөӯм–ҙ Payloadк°Җ м „мІҙ Delta Sizeм—җ лҜём№ҳлҠ” мҳҒн–ҘмқҖ мғқк°ҒліҙлӢӨ нҒ¬м§Җ м•ҠлӢӨ. UTF-8 мҡ°л ӨлҠ” Benchmark Driverм—җ м–ём–ҙлі„ Payload GeneratorлҘј нҸ¬н•Ён•ҳм§Җ м•Ҡмңјл©ҙ кІҖмҰқн•  мҲҳ м—ҶлӢӨ.

  4. Split-brain Detectionмқҳ False Positive Threshold: Section 5.2мқҳ Threshold(SYNC_DEVIATION = 50ms)лҠ” 3-Gateway Mesh кё°мӨҖмқҙлӢӨ. 7-Gateway мқҙмғҒмңјлЎң нҷ•мһҘн•ҳл©ҙ network chaosлЎң мқён•ң transient deviationмқҙ 200msлҘј л„ҳлҠ” кІҪмҡ°к°Җ л°ңмғқн•  мҲҳ мһҲм–ҙ, Dynamic Threshold Calibrationмқҙ н•„мҡ”н•ҳлӢӨ. Auto-Benchmarkк°Җ мқҙ Dynamic Calibrationк№Ңм§Җ мһҗлҸҷнҷ”н•ҙм•ј мӢӨм „м—җм„ң мң мҡ©н•ҳлӢӨ.

  5. WAL Tiered Storage м „нҷҳмқҳ Atomicity: Section 4.3м—җм„ң HotвҶ’Cold м „нҷҳмқ„ м–ёкёүн–Ҳм§Җл§Ң, мқҙ кіјм •м—җм„ңмқҳ Read-Write ConflictлҘј лӢӨлЈЁм§Җ м•Ҡм•ҳлӢӨ. ColdлЎң мқҙлҸҷ мӨ‘мқё WAL Blockм—җ лҢҖн•ң Read мҡ”мІӯмқҙ л“Өм–ҙмҳӨл©ҙ м–ҙл–»кІҢ мІҳлҰ¬н•  кІғмқёк°Җ? Shadow Copy, Copy-on-Write, лҳҗлҠ” Gentle Stop-the-world Compaction мӨ‘ м„ нғқмқҙ н•„мҡ”н•ҳлӢӨ.

  6. PIPA 5л…„ ліҙкҙҖ vs GDPR мһҠнһҗ к¶ҢлҰ¬ (Right to be Forgotten) м¶©лҸҢ: мқҳлЈҢ Federation мӢңлӮҳлҰ¬мҳӨм—җм„ң PIPA 5л…„ ліҙкҙҖмқҖ 1л…„ ліҙкҙҖліҙлӢӨ м—„кІ©н•ҳм§Җл§Ң, GDPR "мһҠнһҗ к¶ҢлҰ¬" мІӯкө¬к°Җ л“Өм–ҙмҳӨл©ҙ м–ҙл–»кІҢ н•  кІғмқёк°Җ? Merkle Hash кё°л°ҳмқҳ Incremental Expunge (WALм—җм„ң нҠ№м • record м ңкұ° + мҰқм Ғ мң м§Җ)к°Җ н•„мҡ”н•ҳм§Җл§Ң, мқҙлҠ” WALмқҳ append-only мҶҚм„ұкіј м¶©лҸҢн•ңлӢӨ. ZKPлЎң "н•ҙлӢ№ recordлҠ” мЎҙмһ¬н•ҳм§Җ м•ҠлҠ”лӢӨ"лҘј мҰқлӘ…н•  мҲҳ мһҲм§Җл§Ң, мқҙлҠ” #069 Observabilityм—җм„ңлҸ„ м–ёкёүн•ң н•ңкі„лӢӨ.

  7. Auto-Benchmarkк°Җ Gateway мһҗмІҙм—җ лҜём№ҳлҠ” л¶Җн•ҳ: Section 3мқҳ Benchmark FrameworkлҠ” м„ұлҠҘ мёЎм •мқ„ мң„н•ҙ Gatewayм—җ м¶”к°Җ л¶Җн•ҳлҘј к°Җн•ңлӢӨ. нҠ№нһҲ Online Monitor Modeм—җм„ң Spike к°җм§Җ мӢң Auto-BenchmarkлҘј Triggerн•ҳл©ҙ, мқҙлҜё Spiking мӨ‘мқё Gatewayм—җ м¶”к°Җ л¶Җн•ҳк°Җ к°Җн•ҙм ём„ң Observability Paradox (кҙҖм°° н–үмң„к°Җ мӢңмҠӨн…ңмқ„ ліҖнҷ”мӢңнӮҙ)к°Җ л°ңмғқн•ңлӢӨ. мқҙ л¬ём ңлҠ” Out-of-band Tracingкіј Adaptive SamplingмңјлЎң мҷ„нҷ”н•ҙм•ј н•ңлӢӨ.

  8. мһ¬л¬ҙ лӘЁлҚё л¶Җмһ¬: Capacity Planningмқҳ м¶ңл ҘмқҖ "Gateway лӘҮ лҢҖ"мҳҖм§Җл§Ң, мӢӨм ң мқҳмӮ¬кІ°м •мһҗлҠ” л№„мҡ© лӘЁлҚё(EC2/GKE л…ёл“ң л№„мҡ©, NVMe vs EBS gp3 л№„мҡ©, Cross-region Data Transfer л№„мҡ©)мқ„ н•Ёк»ҳ лҙҗм•ј н•ңлӢӨ. лІӨм№ҳл§ҲнҒ¬ кІ°кіј вҶ’ л№„мҡ© мҳҲмёЎ вҶ’ CAPEX/OPEX мҠ№мқёмңјлЎң мқҙм–ҙм§ҖлҠ” нҢҢмқҙн”„лқјмқёмқҙ мӢӨм „ Capacity Planningмқҳ мҷ„м „мІҙлӢӨ.
